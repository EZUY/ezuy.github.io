About

I design reproducible pipelines that turn large-scale genomic, transcriptomic, and epigenomic data into biological insight — spanning single-cell and spatial transcriptomics, chromatin profiling, and statistical genetics. Resources

Tools and platforms I recommend for reproducible genomics, with a quick start to get you going. Great for anyone building analysis pipelines or exploring single-cell data.

Nextflow

Docs ↗

A workflow language that makes pipelines reproducible and portable — the same code runs on your laptop, an HPC cluster, or the cloud, with containers handling dependencies.

Quick start

curl -s https://get.nextflow.io | bash
./nextflow run hello

nf-core

Pipelines ↗

A community-curated collection of production-ready Nextflow pipelines (RNA-seq, scRNA-seq, ChIP-seq, and more) — peer-reviewed, tested, and versioned so you don't reinvent the wheel.

Quick start

# run the rnaseq pipeline on the built-in test data
nextflow run nf-core/rnaseq -profile test,docker \
  --outdir results

CZ CELLxGENE

Explore ↗

The Chan Zuckerberg Initiative's platform for exploring single-cell data. Browse millions of annotated cells in the browser, or pull standardized data programmatically via the Census API.

Quick start

pip install cellxgene-census
# then query the Census in Python — see the docs

Seurat & Scanpy

Seurat ↗

The two standard toolkits for single-cell analysis — Seurat in R, Scanpy in Python. QC, normalization, clustering, and visualization for scRNA-seq and multimodal data.

Quick start

# Python
pip install scanpy
# R
install.packages("Seurat")

Bioconductor

Packages ↗

The home of R packages for genomics — DESeq2, edgeR, limma, and hundreds more for bulk and single-cell analysis, all with consistent data structures and rich documentation.

Quick start

if (!require("BiocManager")) install.packages("BiocManager")
BiocManager::install("DESeq2")

Seqera / Wave & Fusion

Seqera ↗

Tooling from the team behind Nextflow — Seqera Platform for launching and monitoring runs, plus Wave containers and the Fusion file system for smoother cloud execution.

# and shared, monitored execution
